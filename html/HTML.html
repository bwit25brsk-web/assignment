HTML

HyperText Markup Language — the skeleton that holds every webpage together.

The Concept

What is HTML?

HTML is structure

HTML (HyperText Markup Language) describes the meaning and organisation of content on a page. It tells the browser what each piece of content is — a heading, a paragraph, a link, a form — not how it looks (that's CSS).

  • Tags wrap content: <p>text</p>
  • Attributes add information: href="", alt=""
  • Semantic tags carry meaning: <nav>, <article>

Webpage structure

Every HTML document follows this pattern:

<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- metadata, title, CSS links -->
    <title>My Page</title>
  </head>
  <body>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
  </body>
</html>

Code Snippet — Form structure

<form action="/submit" method="post">

  <label for="name">Full Name</label>
  <input type="text" id="name" name="name"
         placeholder="Manshi Gyawali" required />

  <label for="email">Email</label>
  <input type="email" id="email" name="email"
         placeholder="you@example.com" required />

  <button type="submit">Submit</button>

</form>

Accessibility

♿ Accessibility decisions

  • Every <input> has a paired <label> — screen readers announce the field name
  • The form uses type="email" so mobile keyboards show the @ key
  • Submit button has a visible, descriptive label — not just an icon

Reflection

💭 Reflection

HTML felt simple at first, but semantic markup — knowing when to use <article> vs <div> — took real thought. The live form demo made the concept click: you can see the markup update as you type. What comes next: adding pattern validation and real error messages.
