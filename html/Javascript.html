JavaScript

The language that makes pages behave — events, DOM, logic, and interactivity.

The Concept

What is JavaScript?

The third layer

HTML structures content, CSS styles it, and JavaScript makes it respond. JS runs directly in the browser — no compilation needed — and can update the page without reloading it.

  • DOM manipulation: change any element on the page
  • Event listeners: react to clicks, keypresses, scrolls
  • Fetch API: talk to servers asynchronously

Dark Mode Toggle

Switch the demo card between light and dark using a single JS class toggle.

🌙 Dark Mode

This card demonstrates a dark mode toggle. The JS adds/removes a class; CSS transitions handle the colour animation smoothly.

Mode: Light

Code Snippet

const toggle = document
  .getElementById('dark-toggle');
const card   = document
  .getElementById('dark-demo');

toggle.addEventListener('change', () => {
  card.classList.toggle(
    'dark-on',
    toggle.checked
  );
});

Accessibility

♿ Accessibility decisions

  • Calculator buttons have aria-label attributes for screen readers (÷ vs "Divide")
  • The display has aria-live="polite" so screen readers announce results
  • The toggle has a visible <label> — not just a visual switch

Reflection

💭 Reflection

JavaScript is where the web comes alive. Building the calculator forced me to think about state — what does the calculator "remember" between button presses? The dark mode toggle taught me that CSS transitions do the heavy lifting; JS just flips a class. What comes next: refactoring the calculator to use a class with proper encapsulation.
